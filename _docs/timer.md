# タイマー

便利ツールカテゴリに「タイマー」を追加。1つのパネル内にタブ切り替えで4種類のタイマーを実装。

## 機能（4タイプ）

| タブ | 内容 |
|---|---|
| カウントダウン | 時/分/秒の指定 ＋ プリセット（1〜60分）。残り時間を円リングで表示。0で終了音＋通知 |
| ストップウォッチ | 経過時間を 1/100 秒まで計測。ラップ記録対応 |
| ポモドーロ | 作業/短い休憩/長い休憩を自動サイクル。完了ポモドーロ数をカウント。「自動で次へ」ON/OFF、スキップ可 |
| 発表タイマー | 経過時間を計測し、1鈴/2鈴/3鈴の各タイミングで合図音（1回/2回/3回）。リング色が緑→黄→赤に変化。本鈴超過は「超過 +mm:ss」表示 |

## 実装ファイル

- `js/tools/timer.js` … 本体ロジック（IIFE、`var`/関数ベースで既存ツールに合わせる）
- `index.html` … サイドバー項目（便利ツール 7→8）、ホームカード、`#tool-timer` パネル、`<script>` 追加
- `css/tools.css` … `.tm-*` スタイル（タブ・SVGリング・プリセット等）を末尾に追加

## 設計上のポイント

- **時刻ベース計算**：残り/経過は `Date.now()` と `endTs`/`startTs` から都度算出する。`setInterval` のカウント加算式にしないことで、タブを非アクティブにしてもズレない（バックグラウンドで `setInterval` が ~1秒間隔にスロットリングされても、復帰時に正しい値を再計算）。
- **アラームのバックグラウンド発火**：更新ループは `requestAnimationFrame`（裏タブで停止）ではなく `setInterval(50ms)` を使用。裏タブでも発火するため、画面を見ていなくても終了音・通知が鳴る。
- **音は Web Audio で生成**：音声ファイル不要。`OscillatorNode` のビープを N 回鳴らす。`AudioContext` はスタートボタン（ユーザー操作）で生成・resume。
- **円リング**：SVG `<circle>` の `stroke-dasharray` / `stroke-dashoffset` で進捗表示。`r` から円周を算出。`transform: rotate(-90deg)` で頂点起点。
- **ブラウザ通知**：`tm-notify` チェック時に `Notification.requestPermission()`。タブ名（`document.title`）に残り時間を表示し、裏タブでも一目で分かる。
- **画面スリープ防止**：対応ブラウザのみ Wake Lock API をベストエフォートで取得（非対応は無視）。
- 動作中の全タイマーが停止したらループ（`setInterval`）とWake Lockを解放。

## 既知の注意点

- ブラウザ通知は HTTPS もしくは `localhost` でのみ許可される（`file://` 直開きだと通知が出ない場合がある）。音とタブ名表示は `file://` でも動作。
- iOS Safari はバックグラウンド時に `setInterval` をほぼ停止するため、裏画面に長く置くと終了音が遅れることがある（前面復帰時に発火）。
